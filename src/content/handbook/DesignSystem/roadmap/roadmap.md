# UX Design 学习路线图：从入门到能做产品体验设计

> 参考来源：roadmap.sh 的 UX Design Roadmap，并结合前端开发者、AI 产品设计、个人项目落地场景重新整理。  
> 目标不是“成为会画界面的美工”，而是建立一套从用户、问题、行为、流程、界面到验证的完整设计能力。

---

## 0. 学习目标

学完这份路线后，你应该能做到：

1. 看懂一个产品的体验问题，而不是只评价“好不好看”。
2. 能从用户目标出发，拆出用户旅程、信息架构和关键流程。
3. 能画出低保真线框图、高保真界面和可交互原型。
4. 能用用户测试、数据和反馈验证设计是否有效。
5. 能把 UX 设计能力接入自己的前端 / AI 应用开发流程。

最终产出建议：

- 一份完整 UX Case Study
- 一个 Figma 原型
- 一个可上线的产品页面或 MVP
- 一份设计说明文档
- 一份用户研究与可用性测试记录

---

## 1. UX Design 是什么

### 1.1 UX 与 UI 的区别

UX Design 关注的是：

- 用户为什么来？
- 用户想完成什么任务？
- 过程中有什么阻碍？
- 产品如何降低理解成本、操作成本和决策成本？
- 用户完成任务后是否满意、是否愿意再次使用？

UI Design 更偏向：

- 页面视觉
- 颜色、字体、间距
- 组件样式
- 视觉层级
- 品牌感

简单说：

> UI 是界面长什么样，UX 是用户用起来顺不顺。

一个优秀产品通常需要二者结合。只会 UI，容易做出“漂亮但不好用”的页面；只会 UX，不懂 UI，容易做出“逻辑正确但没有吸引力”的产品。

### 1.2 UX 的核心工作流

可以理解成这条链路：

```text
理解问题 → 研究用户 → 定义需求 → 设计流程 → 设计界面 → 原型验证 → 迭代优化
```

对应到具体产物：

```text
问题背景 → 用户画像 → 用户旅程 → 信息架构 → Wireframe → Prototype → Usability Test → Design Spec
```

## 2. 第一阶段：用户与问题理解

这一阶段的重点不是画图，而是搞清楚：用户是谁、问题是什么、为什么值得解决。

### 2.1 用户研究基础

需要掌握：

- 用户访谈
- 问卷调研
- 竞品分析
- 场景分析
- 用户画像 Persona
- 用户旅程图 User Journey Map
- 痛点 Pain Point
- 需求 Need
- 动机 Motivation
- 使用场景 Context

你要学会区分：

```text
用户说想要的 ≠ 用户真正需要的
用户抱怨的现象 ≠ 真正的问题根源
功能需求 ≠ 用户目标
```

### 2.2 练习任务

选择一个你熟悉的产品，例如：

- ChatGPT
- B 站
- 小红书
- Notion
- Figma
- 微信文件传输助手
- 你的 AI 图片检测项目 BemoLens

完成以下分析：

```md
## 产品名称

## 目标用户

## 用户核心目标

## 高频使用场景

## 当前体验问题

## 用户真正想完成的任务

## 可能的改进方向
```

### 2.3 AI 辅助提示词

```text
你是一名 UX Researcher。请帮我分析 [产品名称] 的目标用户、核心使用场景、用户痛点、用户任务和可能的体验优化方向。请不要只从功能角度分析，要从用户目标和决策过程分析。
```

## 3. 第二阶段：人类决策与行为设计

roadmap.sh 的 UX Design 路线很强调 Human Decision Making、Nudge Theory、Persuasive Technology、Behavior Design、Behavioral Science、Behavioral Economics 等内容。这一点很重要，因为 UX 本质上不是“摆按钮”，而是在设计用户如何理解、选择和行动。

### 3.1 Human Decision Making

需要理解：

- 用户不是完全理性的
- 用户注意力有限
- 用户容易被默认选项影响
- 用户会受路径、提示、文案、视觉层级影响
- 用户经常使用直觉判断，而不是深度思考

常见概念：

- 认知负荷 Cognitive Load
- 心智模型 Mental Model
- 决策疲劳 Decision Fatigue
- 启发式判断 Heuristics
- 双系统理论 Dual Process Theory
- 系统 1：快速、直觉、自动
- 系统 2：慢速、理性、费力

### 3.2 Nudge Theory 助推理论

Nudge 可以理解为：

> 不强迫用户，但通过环境、默认项、提示、排序、反馈等方式，引导用户做出更好的选择。

常见例子：

- 默认勾选推荐设置
- 把主要按钮放得更明显
- 用进度条提示用户还差几步完成
- 用空状态引导用户创建第一个项目
- 用风险提示帮助用户避免错误操作

注意：助推不是操纵。好的 UX 应该帮助用户完成自己的目标，而不是利用用户弱点。

### 3.3 Persuasive Technology 说服性技术

说服性技术关注：产品如何通过技术影响用户态度或行为。

常见形式：

- 打卡
- 成就徽章
- 连续使用天数
- 进度反馈
- 社交证明
- 推荐系统
- 个性化提醒

需要警惕：

- 过度上瘾设计
- 黑暗模式 Dark Pattern
- 强迫留存
- 诱导消费
- 隐私不透明

### 3.4 Behavior Design 行为设计

你需要理解行为发生通常需要三个因素：

```text
动机 Motivation + 能力 Ability + 触发 Trigger
```

如果用户没有完成某个操作，可能不是“用户懒”，而是：

- 动机不足
- 操作太难
- 没有合适提示
- 不知道下一步
- 不信任产品
- 害怕出错

### 3.5 练习任务

找一个 App 的注册 / 购买 / 上传 / 发布流程，分析：

```md
## 用户目标

## 产品希望用户做什么

## 当前流程中有哪些助推设计

## 哪些地方降低了用户行动能力

## 哪些地方增加了用户认知负担

## 有没有黑暗模式

## 如何改进
```

## 4. 第三阶段：信息架构与用户流程

当你理解用户之后，就要把复杂信息组织成清晰结构。

### 4.1 信息架构 Information Architecture

需要掌握：

- 页面层级
- 导航结构
- 内容分类
- 标签命名
- 搜索与筛选
- 面包屑
- 空状态
- 错误状态
- 设置页结构

核心问题：

```text
用户能不能快速找到自己要的东西？
用户知不知道自己现在在哪里？
用户知不知道下一步该做什么？
```

### 4.2 用户流程 User Flow

你需要能画出：

```text
入口 → 操作 → 系统反馈 → 决策点 → 成功 / 失败 → 下一步
```

例如 AI 图片检测产品：

```text
进入首页 → 上传图片 → 检查格式 → 分析中 → 展示结论 → 展开证据 → 分享 / 重试
```

### 4.3 练习任务

为 BemoLens 画一份用户流程：

```md
## 主流程
上传图片 → 等待分析 → 查看结论 → 查看证据 → 重试 / 分享

## 异常流程
图片过大 → 提示压缩
格式不支持 → 提示支持格式
无法判断 → 展示不确定状态
网络失败 → 允许重试

## 关键决策点
用户是否信任检测结论？
用户是否愿意继续查看证据？
用户是否愿意分享结果？
```

## 5. 第四阶段：线框图、原型与交互设计

### 5.1 Wireframe 低保真线框图

低保真阶段不要急着纠结颜色和字体，重点是：

- 页面结构
- 内容优先级
- 操作路径
- 信息分组
- 按钮位置
- 状态变化

建议工具：

- Figma
- FigJam
- Whimsical
- Excalidraw
- 纸笔

### 5.2 Prototype 原型

原型分为：

- 低保真原型：验证流程
- 中保真原型：验证信息结构和交互
- 高保真原型：接近真实产品

你要学会设计：

- 点击跳转
- 弹窗
- 抽屉
- Popover
- Toast
- Loading
- Empty State
- Error State
- Success State
- Onboarding

### 5.3 Interaction Design 交互设计

需要关注：

- 操作是否可预期
- 反馈是否及时
- 状态是否清晰
- 错误是否可恢复
- 是否符合用户心智模型
- 是否减少不必要步骤

常见原则：

```text
看得见状态
知道下一步
犯错能恢复
重要操作要确认
高频操作要更短路径
危险操作要更谨慎
```

### 5.4 练习任务

任选一个功能，完成三版设计：

```text
版本 1：低保真线框图
版本 2：中保真交互图
版本 3：高保真视觉稿
```

推荐题目：

- AI 图片上传检测页
- 个人博客文章发布页
- AI Agent 任务管理页
- 模型 API 消耗统计页
- 简历投递追踪页

## 6. 第五阶段：视觉设计与 UI 基础

UX 不等于 UI，但 UX 设计师不能完全不懂 UI。

### 6.1 视觉层级

需要掌握：

- 字号层级
- 字重层级
- 间距层级
- 颜色层级
- 卡片层级
- 主次按钮
- 内容密度

判断标准：

```text
用户第一眼看到什么？
第二眼看到什么？
最重要的操作是否足够明显？
页面有没有噪音？
```

### 6.2 Design System 设计系统基础

需要理解：

- Color Token
- Typography Token
- Spacing Token
- Radius Token
- Shadow Token
- Component Variant
- Button
- Input
- Modal
- Tabs
- Card
- Badge
- Toast
- Form

如果你是前端开发者，设计系统尤其重要，因为它能直接映射到代码。

### 6.3 可访问性 Accessibility

必须掌握：

- 对比度
- 键盘可操作
- 表单 label
- 错误提示
- 焦点状态
- 语义化 HTML
- 屏幕阅读器基础

最低要求：

```text
不要只靠颜色表达状态
文字对比度足够
按钮和输入框有明确状态
错误提示要具体
表单要能被键盘操作
```

## 7. 第六阶段：可用性测试与验证

设计不是靠感觉定输赢，而是要验证。

### 7.1 Usability Testing 可用性测试

测试时关注：

- 用户是否能完成任务
- 用户在哪里卡住
- 用户是否理解文案
- 用户是否误点
- 用户是否产生不信任
- 用户完成任务用了多久

基础测试模板：

```md
## 测试目标

## 测试用户

## 测试任务

## 观察记录

## 用户卡点

## 用户原话

## 问题严重程度

## 改进建议
```

### 7.2 指标意识

UX 常见指标：

- 任务完成率
- 首次成功率
- 完成时间
- 错误率
- 跳出率
- 转化率
- 留存率
- NPS
- SUS 可用性量表

注意：

```text
数据告诉你哪里有问题，不能直接告诉你为什么有问题。
用户访谈告诉你为什么，但不一定代表所有人。
最好结合定量和定性。
```

## 8. 第七阶段：UX 写作与产品表达

很多体验问题不是界面问题，而是文案问题。

### 8.1 UX Writing

需要掌握：

- 按钮文案
- 表单提示
- 错误提示
- 空状态文案
- 加载文案
- 成功提示
- 风险提示
- 新手引导

好的 UX 文案应该：

```text
清楚
具体
可行动
不吓人
不废话
符合用户当前情绪
```

### 8.2 示例

差的文案：

```text
发生错误
```

更好的文案：

```text
图片上传失败，可能是文件过大。请压缩到 10MB 以下后重试。
```

差的按钮：

```text
确定
```

更好的按钮：

```text
开始检测
确认删除
重新上传
保存草稿
```

## 9. 第八阶段：AI 产品时代的 UX

你现在关注 AI 应用，所以需要额外补一层 AI UX。

### 9.1 AI 产品的特殊体验问题

AI 产品常见问题：

- 输出不稳定
- 用户不知道 AI 正在做什么
- 用户不知道结果是否可信
- 用户不知道如何纠错
- 用户不知道如何写 Prompt
- 系统边界不清晰
- 等待时间长
- 错误不可解释

### 9.2 AI UX 设计重点

需要设计：

- 能力边界说明
- 输入示例
- Prompt 模板
- 生成过程反馈
- 结果置信度
- 证据展开
- 可编辑结果
- 重新生成
- 人工修正
- 失败兜底

### 9.3 结合 BemoLens 的例子

AI 图片检测结果页不要只说：

```text
该图片可能是 AI 生成
```

更好的结构是：

```text
结论：疑似 AI 生成
可信度：中等
原因：检测到 C2PA 元数据 / 视觉异常 / EXIF 缺失
限制：该判断不能作为绝对证据
下一步：查看证据 / 重新上传 / 分享报告
```

## 11. 推荐练习项目

### 项目 1：重设计一个上传流程

目标：提升上传成功率和用户信任。

你需要设计：

- 上传前说明
- 拖拽上传区
- 格式限制
- 上传中状态
- 失败提示
- 成功状态
- 重新上传

### 项目 2：重设计一个设置页

目标：让复杂配置更容易理解。

你需要设计：

- 分组结构
- 默认选项
- 风险提示
- 保存反馈
- 重置功能

### 项目 3：AI Agent 任务执行面板

目标：让用户理解 Agent 正在做什么。

你需要设计：

- 任务列表
- 当前步骤
- 工具调用记录
- 失败重试
- 人工接管
- 结果摘要

### 项目 4：AI 图片检测报告页

目标：让用户信任但不过度迷信检测结果。

你需要设计：

- 总结论
- 置信度
- 证据层级
- 风险提示
- 限制说明
- 分享报告

## 12. UX Case Study 模板

```md
# 项目名称

## 1. 项目背景

这个项目解决什么问题？为什么值得解决？

## 2. 我的角色

我负责了哪些部分？研究、流程、原型、视觉、测试、开发？

## 3. 目标用户

用户是谁？他们在什么场景下使用？

## 4. 问题定义

用户遇到的核心问题是什么？

## 5. 研究过程

做了哪些访谈、竞品分析或观察？

## 6. 关键洞察

发现了什么？哪些发现改变了你的设计方向？

## 7. 设计目标

这次设计希望改善什么？

## 8. 用户流程

展示主流程和异常流程。

## 9. 方案探索

展示几个方案，并说明为什么选择最终方案。

## 10. 原型设计

展示低保真、中保真、高保真。

## 11. 可用性测试

测试了什么？发现了什么问题？如何修改？

## 12. 最终方案

展示最终界面和关键交互。

## 13. 结果与反思

有没有数据？有没有用户反馈？下一步怎么优化？
```

## 13. 常用 AI 提示词

### 13.1 做用户研究

```text
你是一名资深 UX Researcher。请帮我围绕 [产品/功能] 设计一份用户研究计划，包括研究目标、目标用户、访谈问题、观察重点和最终产出。
```

### 13.2 做竞品分析

```text
请从 UX 角度分析 [竞品 A]、[竞品 B]、[竞品 C]。重点比较它们的用户流程、信息架构、关键页面、转化设计、可用性问题和可借鉴点。
```

### 13.3 优化用户流程

```text
下面是我的产品流程：[粘贴流程]。请帮我找出其中的认知负担、重复步骤、信任断点、异常流程缺失，并给出优化后的用户流程。
```

### 13.4 生成线框图说明

```text
请为 [功能名称] 生成一份低保真线框图说明，要求包含页面结构、内容层级、主要操作、异常状态和交互说明。
```

### 13.5 做可用性测试

```text
请为 [产品/页面] 设计一份可用性测试方案，包括测试目标、测试任务、观察指标、访谈问题和结果记录模板。
```

### 13.6 写 UX Case Study


## 16. 给前端开发者的特别建议

你不需要一开始就追求成为纯 UX Designer。更现实的方向是：

```text
前端开发 + UX 思维 + AI 产品能力
```

这会让你比普通前端更有竞争力，因为你不仅能实现页面，还能判断：

- 这个功能为什么要做？
- 用户是否真的需要？
- 这个流程是否合理？
- 页面状态是否完整？
- 用户是否能理解 AI 的输出？
- 设计能不能被工程稳定实现？

你的最佳训练方式不是临摹 100 个 Dribbble 页面，而是不断重构自己的项目体验。

推荐你从 BemoLens 开始，把它当作 UX 训练项目：

```text
上传页 → 检测中 → 结果页 → 证据展开 → 不确定状态 → 分享报告
```

把每个页面都补齐：

- 用户目标
- 页面任务
- 信息层级
- 交互状态
- 异常状态
- 文案策略
- 可用性测试问题
- 前端组件拆分

这样学 UX，会比单纯看课有效得多。
